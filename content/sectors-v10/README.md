# نظام القطاعات التحريرية v10

هذا المجلد هو مصدر المحتوى للقطاعات الجديدة. لا تعدّل القوالب عند إضافة مقال؛ عدّل ملفات JSON فقط.

## الملفات

- `family.json` — الصحة النفسية للأسرة.
- `child.json` — الصحة النفسية للطفل.
- `home.json` — الصحة النفسية للعائلة والبيت.
- `women.json` — الصحة النفسية للمرأة.
- `tips.json` — أدلة النصائح العملية.

## إضافة مقال

أضف كائنًا داخل `articles` بالحقول:

```json
{
  "slug": "stable-english-slug",
  "title": "عنوان عربي واضح",
  "summary": "ملخص أصلي يحدد المشكلة وفائدة الصفحة.",
  "signals": ["ثلاث ملاحظات على الأقل"],
  "steps": ["أربع خطوات عملية على الأقل"],
  "phrases": ["عبارتان تساعدان في الحوار"],
  "avoid": "سلوك أو خطأ ينبغي تجنبه"
}
```

## قواعد الجودة

1. العنوان والـ slug فريدان.
2. لا تُنشأ صفحة بلا ملاحظات وخطوات عملية.
3. لا تُستخدم لغة تشخيص جازمة.
4. تُذكر حالات الخطر وطلب الدعم عند الحاجة.
5. المصادر الرسمية تحفظ في مصفوفة `sources` لكل قطاع.
6. يشغّل GitHub Actions التحقق والبناء تلقائيًا.
7. أي إضافة جديدة تدخل تلقائيًا في البحث وملفات API وCSV وخريطة الموقع.

## تشغيل محلي

```bash
python scripts/design_content_v10.py
```

يجب تشغيله بعد بناء الإصدار التاسع.
