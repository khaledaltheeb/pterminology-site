# خط الأساس النشط لقاعدة البيانات

**الملف الوحيد القابل للتنفيذ حاليًا:** `postgresql.v0.2.0.sql`

الملفات ذات الإصدار `0.1.x` سجلات تاريخية لمسودات اكتشفت فيها مشكلات أثناء المراجعة. يمنع تنفيذها أو دمجها في سلسلة ترحيل. المرجع الآلي للحالة هو `MIGRATION_MANIFEST.json`.

## طريقة التحقق

```bash
psql --set ON_ERROR_STOP=1 --file provider-assessment-platform/database/postgresql.v0.2.0.sql
psql --set ON_ERROR_STOP=1 --file provider-assessment-platform/database/tests/rls-smoke.sql
```

تنفذ أوامر الاختبار على قاعدة بيانات مؤقتة وببيانات صناعية فقط. لا يعتبر نجاحها اعتمادًا إنتاجيًا.

## ما يثبته الاختبار

- قبول PostgreSQL للمخطط والسياسات والمشغلات نحويًا.
- عزل المؤسسة.
- عزل الحالة داخل المؤسسة بحسب التكليف.
- انتقال الحالة إلى `safety_hold` عند حدث عاجل.
- منع تعديل سجل سلامة غير قابل للتغيير.
- وجود مفاتيح أجنبية مركبة للمؤسسة والحالة.

## ما لا يثبته

- سلامة المنظومة أمام اختبار اختراق كامل.
- الامتثال القانوني في الأردن أو أي سوق آخر.
- ملاءمة سياسة الاحتفاظ والنسخ الاحتياطي.
- قابلية التوسع والأداء تحت الحمل.
- صحة تكامل ناشر أداة تجارية.
- اعتماد المنصة للاستخدام السريري.

## قبل الإنتاج

يلزم على الأقل: مراجعة أمن مستقلة، DPIA، نموذج تهديد، اختبار اختراق، إدارة مفاتيح، نسخ احتياطي واستعادة، مراقبة، فصل أدوار، مراجعة قانونية، وتجربة ميدانية مراقبة.
