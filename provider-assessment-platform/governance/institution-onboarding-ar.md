# اعتماد المركز أو المؤسسة قبل استخدام منصة التقييم

> لا يتيح توقيع عقد تجاري وحده الوصول إلى بيانات أو أدوات أو تقارير. تفعيل المؤسسة قرار منفصل يحتاج اجتياز بوابات قانونية وأمنية وسريرية وتشغيلية.

## 1. حالات المؤسسة

| الحالة | المعنى |
|---|---|
| `applicant` | طلب أولي دون وصول للنظام |
| `due_diligence` | جمع وثائق والتحقق منها |
| `configuration` | إعداد الهوية والأدوار والطوارئ والخصوصية ببيانات صناعية |
| `pilot_synthetic` | تدريب وتجربة ببيانات صناعية فقط |
| `pilot_restricted` | تجربة ميدانية محدودة بعد الموافقات المستقلة |
| `active` | تشغيل ضمن النطاق والأدوات المعتمدة فقط |
| `restricted` | منع وظائف أو أدوات محددة مع استمرار الحد الأدنى الآمن |
| `suspended` | إيقاف المؤسسة عن بدء عمليات جديدة |
| `terminated` | إنهاء العلاقة وفق خطة تصدير واحتفاظ وحذف |

لا تنتقل المؤسسة تلقائيًا بين الحالات بناء على دفع رسوم أو إكمال نموذج.

## 2. ملف المؤسسة المطلوب

### الهوية والصفة القانونية

- الاسم القانوني والتجاري.
- رقم التسجيل والبلد والعنوان القانوني.
- الأشخاص المخولون بالتوقيع.
- نطاق الخدمات والفئات العمرية.
- تراخيص المركز والجهات الرقابية المختصة.
- التأمين المهني أو ما يقوم مقامه عند اللزوم.
- أي إجراءات أو قيود أو شكاوى جوهرية يجب الإفصاح عنها قانونيًا.

### الحوكمة السريرية

- المدير السريري ونائبه.
- آلية اعتماد التشخيصات والتقارير.
- سياسة الإشراف على المبتدئين والمتدربين.
- لجنة جودة أو مراجعة حالات معقدة.
- سياسة حماية الطفل والأشخاص المعرضين للخطر.
- سياسة الشكاوى والتصحيح وسحب التقارير.
- مسار الإحالة الطبية والنفسية والسمعية والبصرية والتعليمية.
- قائمة الخدمات التي لن يقدمها المركز.

### الخصوصية والقانون

- تحديد المتحكم والمعالج لكل تدفق.
- اتفاق معالجة البيانات والعقود الفرعية.
- DPIA معتمد.
- سياسة الخصوصية والاحتفاظ والحذف وحقوق الأشخاص.
- خريطة نقل البيانات والدول والمعالجين الفرعيين.
- مسؤول الخصوصية أو جهة الاتصال.
- خطة التعامل مع طلبات الحقوق والشكوى.
- سياسة البحث والتسويق والذكاء الاصطناعي.

### الأمن التقني

- هوية موحدة وMFA.
- إدارة أجهزة وتحديثات وتشفير شاشة وقرص.
- منع الحسابات المشتركة.
- شبكة واتصال آمنان.
- إدارة موظفين عند الانضمام والمغادرة.
- تدريب تصيد واحتيال وتسريب.
- قناة آمنة للملفات والتقارير.
- خطة حادث واستمرارية ونسخ احتياطي.
- اختبار اختراق أو تقييم أمني مناسب للمخاطر.

## 3. فحص الأشخاص والأدوار

لا ينشأ حساب فعّال قبل توثيق:

- الهوية المهنية.
- علاقة العمل بالمؤسسة.
- المسمى والنطاق المسموح.
- الترخيص المهني وحالته وتاريخه وجهة التحقق.
- التدريب العام للمنصة.
- التدريب الخاص بكل أداة أو وظيفة.
- اللغات والكفاءة في التقييم بها.
- الإشراف المطلوب والقيود.
- MFA وقبول سياسات الاستخدام والسرية.

### فصل الأدوار

- مسؤول النظام لا يصبح مراجعًا سريريًا تلقائيًا.
- من أنشأ الخطة لا يوقع التقرير إذا تطلبت سياسة المخاطر مراجعة مستقلة.
- الدعم التقني لا يقرأ المحتوى السريري افتراضيًا.
- المدقق المؤسسي يحتاج نطاقًا زمنيًا وسببًا مسجلًا.
- مدير المؤسسة لا يستطيع منح نفسه صلاحية أداة تجارية دون تحقق مستقل.

## 4. إعداد السلامة المحلية

يمنع تفعيل المؤسسة حتى توثيق واختبار:

- أرقام وخدمات الطوارئ الفعلية في منطقة العمل.
- أقرب مسارات طبية ونفسية وحماية طفل.
- مسؤول الاستجابة داخل ساعات العمل وخارجها.
- قواعد التعامل مع إيذاء النفس والآخرين والإساءة والإهمال.
- فقد المهارات أو التدهور العصبي أو صعوبات البلع والتنفس.
- آلية التسليم والتأكد من استلام الجهة الأخرى.
- اللغة المناسبة للاتصال بالأشخاص والأسر.
- معالجة الحالات عن بعد أو خارج المنطقة.

يجب اختبار سيناريوهات الطوارئ ببيانات صناعية وتوثيق النتائج.

## 5. الأدوات والتراخيص

لكل أداة على حدة:

- اسم الناشر والإصدار واللغة.
- عقد الشراء أو الإذن والرقمنة.
- حق التطبيق عن بعد أو الإلكتروني.
- مواقع وبلدان الاستخدام المسموحة.
- المؤهلات المطلوبة.
- التدريب والشهادة والمدة.
- عدد الاستخدامات أو آلية الرسوم.
- طريقة حفظ البنود والنتائج.
- سياسة الناشر للتقارير والتصدير.
- تاريخ الانتهاء أو المراجعة.
- جهة الاتصال عند الحادث أو كشف المواد.

حالة المؤسسة `active` لا تفعّل جميع الأدوات؛ كل أداة لها تفويض مستقل.

## 6. اختبار القبول المؤسسي

ينفذ فريق المؤسسة ببيانات صناعية حالات تشمل:

1. إنشاء حالة وموافقة صحيحتين.
2. رفض حالة دون موافقة أو سؤال إحالة.
3. منع مقدم غير مكلف من رؤية الحالة.
4. ظهور خطر عاجل وإيقاف المسار والتسليم.
5. تخطيط أداة غير مرخصة ورفض بدء الجلسة.
6. انتهاء تدريب مقدم بين التخطيط والجلسة.
7. نتيجة غير صالحة وانحراف يمنع التفسير المعياري.
8. تعارض بين مصدرين يوقف الخلاصة.
9. إنشاء مسودة تقرير دون السماح بتوقيع غير المخول.
10. تصحيح تقرير بإصدار جديد دون محو القديم.
11. طلب وصول أو تصحيح أو سحب موافقة.
12. حادث تسريب تجريبي وتطبيق خطة الاستجابة.

كل سيناريو يحتاج دليل نجاح ومراجعًا وموعد إعادة اختبار.

- موعد المراجعة التالي.
- مؤشرات الإيقاف الفوري.

## 8. المراقبة المستمرة

### يوميًا أو آليًا

- الحسابات المعطلة أو محاولات الدخول غير العادية.
- انتهاء ترخيص أو تدريب أو حق أداة.
- عمليات تصدير أو تدقيق واسعة.
- حالات خطر دون تسليم مؤكد.
- تقارير موقعة خارج نطاق الدور.
- أدوات أو مسارات معلقة ما زالت مستخدمة.

### شهريًا

- مراجعة المستخدمين والتكليفات.
- عينات من اكتمال الموافقة والمصادر والتقارير.
- الحوادث والشكاوى والأخطاء والتصحيحات.
- اختلاف النتائج بين الفئات واللغات والمراكز.
- زمن الإحالة والتقييم والتقرير دون تشجيع السرعة على حساب الجودة.

### سنويًا أو عند تغير جوهري

- إعادة اعتماد المؤسسة والمستخدمين والأدوات.
- مراجعة DPIA ونموذج التهديد.
- اختبار طوارئ وحادث واستعادة.
- اختبار إتاحة ومشاركة مستخدمين ذوي إعاقة.
- مراجعة العقود والمعالجين الفرعيين.

## 9. التعليق أو الإيقاف

أسباب التعليق تشمل:

- انتهاء ترخيص المركز أو المختص أو الأداة.
- فقد الحوكمة السريرية أو مسؤول السلامة.
- خرق بيانات أو أمن اختبار غير محتوى.
- استخدام النظام للتشخيص أو الأهلية الآلية.
- مشاركة بيانات مع خدمة غير معتمدة.
- رفض التعاون مع تدقيق أو تصحيح جوهري.
- نمط تقارير أو نتائج غير موثوق أو تمييزي.
- تكرار تجاوز الموافقة أو التكليف بالحالة.

يمكن أن يكون الإيقاف لأداة أو مستخدم أو وظيفة أو مؤسسة كاملة حسب الخطر.

## 10. إنهاء العلاقة

قبل الإنهاء:

- تحديد المتحكم القانوني بالملفات.
- إيقاف إنشاء حالات وجلسات جديدة.
- إكمال أو نقل الحالات الجارية بأمان.
- تصدير البيانات المسموح بها بصيغة موثقة ومشفرة.
- إبطال الحسابات والتكاملات والمفاتيح.
- استعادة أو حذف مواد الناشرين وفق العقد.
- تطبيق الاحتفاظ والحذف القانونيين مع سجل إثبات.
- استمرار الوصول إلى سجل التدقيق وفق الالتزام.
- إخطار الأشخاص عندما يلزم.
- معالجة الشكاوى والتقارير المتنازع عليها.

لا يحذف السجل السريري أو التقرير لمجرد انتهاء العقد إذا كان القانون أو سلامة الشخص يتطلب الاحتفاظ به.
