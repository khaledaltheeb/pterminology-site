## 2. سؤال الاستخدام

اكتب السؤال الذي ستجيب عنه الأداة داخل المسار:

```text
PENDING
```

واكتب صراحة ما لا تستطيع إثباته:

```text
PENDING
```

يرفض التفعيل إذا كان الغرض المقترح أوسع من دليل الأداة أو أدلة صدقها.

## 3. الملكية والترخيص

يجب توفير دليل قابل للتحقق لكل بند:

- [ ] مالك حقوق الأداة معروف.
- [ ] النسخة والإصدار المحددان مشمولان.
- [ ] حق الاستخدام المهني موثق.
- [ ] حق الرقمنة أو التطبيق الإلكتروني موثق.
- [ ] التطبيق عن بعد مشمول عند استخدامه.
- [ ] اللغة أو الترجمة مشمولة.
- [ ] دول ومواقع الاستخدام مشمولة.
- [ ] حق تخزين النتيجة أو تصديرها أو إدخالها في التقرير موثق.
- [ ] قيود العرض والطباعة والتسجيل والتقاط الشاشة موثقة.
- [ ] رسوم الاستخدام والحدود والتجديد موثقة.
- [ ] تاريخ الانتهاء والتنبيه قبله معروفان.
- [ ] جهة اتصال الناشر للحوادث معروفة.

**قرار افتراضي:** عند غياب دليل واضح، تكون الأداة `disabled`.

## 4. أمن الاختبار

حدد أين توجد:

- البنود والتعليمات.
- مفاتيح التصحيح.
- جداول التحويل والمعايير.
- ملفات التدريب والإجابات النموذجية.
- النتائج الخام والمعدلة.

الضوابط الدنيا:

- لا تُحفظ المواد المحمية في المستودع العام أو قاعدة العمل السريري.
- لا تظهر في سجل أخطاء أو تحليلات أو نسخ احتياطية غير مرخصة.
- لا يستطيع مسؤول النظام قراءتها افتراضيًا.
- لا تصل إلى نموذج ذكاء اصطناعي إلا بموافقة ناشر صريحة ونطاق معتمد.
- لا يسمح بتنزيلها أو طباعتها إلا إذا سمح العقد.
- كل فتح أو تصدير أو طباعة يسجل عند اشتراط المخاطر أو العقد.

## 6. السكان واللغة والمعايير

وثق:

- العمر الزمني والنمائي المقبول.
- الحالات أو المجموعات التي درست عليها الأداة.
- اللغة الأصلية والترجمة الرسمية.
- عينة التقنين: الدولة، الحجم، العمر، التعليم، الجنس، اللغة، الإعاقة عند توفرها.
- تاريخ جمع المعايير ومدى ملاءمتها الآن.
- هل توجد معايير أردنية أو عربية مناسبة؟
- هل يسمح الدليل باستخدام معايير بديلة؟
- ما القيود على ثنائيي اللغة أو مستخدمي لغة الإشارة أو AAC؟
- ما التكييفات المسموحة وما أثرها في الدرجة؟

### قاعدة الدرجات

- لا تعرض درجة معيارية إذا لم تكن النسخة واللغة وطريقة التطبيق والسكان متوافقة مع دليلها.
- الترجمة الفورية أو غير المقننة لا تنتج درجة معيارية معتمدة.
- عند غياب المعايير الملائمة، يستخدم وصف نوعي أو مقارنة داخلية فقط إذا سمح العلم والغرض، مع إفصاح واضح.

## 8. قواعد التطبيق

يجب ترميز ما يسمح الدليل بترميزه فقط:

- قواعد البداية والعكس والتوقف.
- الحد الأدنى للبنود الصالحة.
- التعامل مع الانقطاع أو عدم الاستجابة.
- الزمن والبيئة والأدوات المساعدة.
- المجيب المناسب وفترة الملاحظة.
- التعديلات والتكييفات المسموحة.
- أسباب إيقاف الجلسة.
- حالات النتيجة: صالحة، مقيدة، غير صالحة.
- فصل الدرجة الخام عن التحويل والتفسير.

لا ينشأ منطق جديد لتسهيل البرمجة إذا لم يدعمه الدليل.

## 9. التصحيح والتحويل

قبل التفعيل:

1. أنشئ حالات مرجعية قانونية من الناشر أو دليل مرخص.
2. احسب يدويًا وبالنظام بصورة مستقلة.
3. اختبر الحدود والقيم المفقودة وقواعد التوقف.
4. اختبر كل جدول عمر ولغة وإصدار مسموح.
5. قارن النتائج حرفيًا مع المرجع.
6. راجع التقريب والنسب المئوية والفواصل الزمنية.
7. اختبر أن الإصدار الخطأ أو العمر الخطأ يرفضان.
8. اختبر أن التعديل غير المعياري يغير حالة الصلاحية.

يحتاج كل تغيير في خوارزمية التصحيح إلى إصدار واختبار انحدار ومراجعة مستقلة.

## 10. الربط بالمسار

لكل نتيجة أو حالة، وثق:

- العقدة التي تستخدم الأداة.
- هل هي شرط أم دليل مساند؟
- المصدر الذي يدعم الانتقال.
- ما الأدلة الأخرى المطلوبة؟
- ما الذي يحدث عند التعارض؟
- ما الذي يحدث عند البيانات المفقودة؟
- هل الانتقال يحتاج تأكيدًا بشريًا؟
- ما الحالات التي تمنع التفسير؟

**يحظر:** `نتيجة مرتفعة → تشخيص`.

الصيغة المقبولة مثلًا:

```text
نتيجة مسح ذات صلاحية مقبولة
+ تاريخ يدعم المؤشر
+ أثر وظيفي أو سبب إحالة
→ تقييم تخصصي إضافي محدد
→ مراجعة بشرية
```

## 11. اختبار الوصول والإتاحة

- قارئ الشاشة وترتيب القراءة.
- لوحة المفاتيح والتركيز.
- تكبير النص والتباين.
- عدم الاعتماد على اللون أو الصوت وحده.
- الترجمة النصية والتعليمات المبسطة عند السماح.
- الوقت الإضافي أو وسيلة الإدخال عندما يسمح الدليل.
- توثيق أثر التكييف في الصلاحية.
- حماية أمن الاختبار مع توفير التكييف المعقول.

لا يعد تعديلًا في الواجهة محايدًا إذا غيّر ترتيب البنود أو طريقة تقديمها أو الاستجابة.

## 12. اختبارات الفشل

يجب أن تفشل الأداة بأمان عند:

- انتهاء ترخيص المؤسسة أو المستخدم.
- غياب تدريب أو دور مطلوب.
- عمر أو لغة أو بلد خارج النطاق.
- عدم وجود موافقة أو خطة حالة.
- خطر عاجل مفتوح.
- نسخة أداة غير متطابقة.
- نسبة بنود مفقودة أكبر من المسموح.
- تعارض بين النتيجة وصلاحية التطبيق.
- توقف تكامل الناشر.
- محاولة إعادة استخدام رمز جلسة أو فحص وصول قديم.
- محاولة استخراج مواد محمية من API أو سجل أو تقرير.

## 15. التعليق والتقاعد

تعلق الأداة فورًا أو جزئيًا عند:

- انتهاء أو نزاع الترخيص.
- اكتشاف خطأ تصحيح أو معيار أو ربط مسار.
- تحديث ناشر يلغي النسخة أو الاستخدام.
- خرق أمن اختبار.
- نتائج تمييزية أو غير متوقعة.
- تغير المتطلبات المهنية أو القانونية.
- فشل تكامل يؤثر في سلامة النتيجة.

عند التعليق:

1. امنع جلسات جديدة.
2. حدد الحالات والتقارير التي استخدمت النسخة.
3. يراجع المختصون الأثر.
4. لا تحذف النتائج أو التقارير القديمة.
5. أصدر تصحيحًا أو إعادة تقييم عند الحاجة.
6. أخطر المؤسسات والمستخدمين والناشر وفق الخطة.
7. لا تعاد الأداة إلا بعد اختبار وموافقة جديدة.
