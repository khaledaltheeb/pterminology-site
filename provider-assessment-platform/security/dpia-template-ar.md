## 2. وصف المعالجة

### الغرض المحدد

صف بصورة دقيقة:

- من يستخدم المنصة؟
- لأي سؤال إحالة أو خدمة؟
- ما القرارات التي تدعمها المنصة؟
- ما القرارات التي تبقى بشرية؟
- هل توجد أدوات تجارية أو تكاملات خارجية؟
- هل يستخدم الذكاء الاصطناعي؟ وفي أي مرحلة؟

**يحظر وصف الغرض بعبارة عامة مثل «تحسين الخدمات» دون تحديد.**

### العمليات

ضع علامة أمام كل عملية مطبقة:

- [ ] إنشاء ملف حالة.
- [ ] تخزين مرجع هوية منفصل.
- [ ] تسجيل الموافقة أو الأساس القانوني.
- [ ] جمع التاريخ النمائي والطبي والتعليمي والاجتماعي.
- [ ] جمع معلومات الأسرة أو المدرسة أو العمل.
- [ ] تشغيل مسار دعم قرار.
- [ ] تطبيق أداة داخل المنصة.
- [ ] استيراد نتيجة من جهة خارجية.
- [ ] تسجيل ملاحظة مباشرة أو ملف صوت/فيديو.
- [ ] إنشاء مسودة تقرير.
- [ ] توقيع تقرير مهني.
- [ ] تصدير أو مشاركة تقرير.
- [ ] متابعة النتائج.
- [ ] تحليل جودة مجهّل.
- [ ] بحث أو تدريب نموذج.

أي عملية غير محددة هنا تحتاج تقييمًا وتحديثًا قبل تنفيذها.

### بيانات ممنوعة افتراضيًا

- بنود الاختبارات ومفاتيح التصحيح وجداول المعايير خارج حدود الناشر المرخصة.
- بيانات لا تخدم سؤال الإحالة أو التزامًا واضحًا.
- نسخ كاملة من وثائق هوية عندما يكفي التحقق والمرجع.
- أسرار أو كلمات مرور أو رموز جلسة.
- بيانات لا يمكن تحديد مسؤوليتها ومدة الاحتفاظ بها.

## 5. خريطة تدفق البيانات

أنشئ رسمًا يوضح لكل تدفق:

1. المصدر.
2. نقطة الإدخال.
3. الخدمة المعالجة.
4. قاعدة أو مخزن البيانات.
5. التكامل الخارجي.
6. البلد أو المنطقة التي تعالج فيها البيانات.
7. التشفير وهوية المستلم.
8. مدة الاحتفاظ والحذف.

مثال مبدئي:

```text
الأسرة/المختص
  → بوابة المؤسسة
  → خدمة الهوية والموافقة
  → مخزن الهوية المنفصل

مقدم الخدمة
  → API مهني
  → محرك المسار والصلاحيات
  → قاعدة العمل السريري
  → حدود ناشر مرخصة عند الحاجة
  → تقرير مشفر وموقع
```

## 7. الضرورة والتناسب

أجب بالأدلة:

- لماذا لا يمكن تحقيق الغرض ببيانات أقل؟
- لماذا يلزم الاحتفاظ بكل حقل؟
- هل يمكن استخدام مرجع بديل عن الهوية؟
- هل يمكن الاستغناء عن الصوت أو الفيديو؟
- هل يمكن تقليل عدد الأشخاص الذين يرون التقرير؟
- هل يمكن الاكتفاء بنتيجة خارجية بدل نسخ مادة محمية؟
- هل كل توصية أو قاعدة مرتبطة بالغرض؟
- هل الاحتفاظ متناسب مع القانون وحاجة الخدمة؟

كل إجابة «لا نعرف» تتحول إلى إجراء قبل الإطلاق.

## 8. حقوق الأشخاص

حدد آلية واضحة لـ:

- الإعلام والشفافية بلغة مفهومة.
- الوصول إلى البيانات.
- تصحيح بيانات أو تقرير دون محو السجل الأصلي.
- الاعتراض أو تقييد معالجة محددة.
- سحب الموافقة حيث تنطبق.
- الحصول على نسخة قابلة للقراءة.
- معرفة الجهات التي استلمت البيانات.
- تقديم شكوى.
- مراجعة قرار مؤثر بواسطة إنسان.
- تكييف الطلب للأشخاص ذوي الإعاقة أو محدودية اللغة.

وثق المهل، والجهة المسؤولة، وطريقة التحقق من الهوية دون جمع زائد.

## 11. حماية الأطفال والأشخاص المعرضين للخطر

- واجهة شفافية مناسبة للعمر والفهم.
- موافقة ولي الأمر حيث تلزم، مع إشراك الطفل أو الشخص واحترام إرادته قدر الإمكان.
- أقل قدر من التسجيلات والوسائط.
- مسار حماية واضح عند الاشتباه بالإساءة أو الاستغلال.
- عدم إرسال تنبيه قد يعرض الشخص لخطر إذا كان الجهاز أو البريد مراقبًا.
- قيود إضافية على البحث والتسويق والتصنيف الآلي.
- آلية بشرية للاعتراض على استنتاج أو تقرير.
