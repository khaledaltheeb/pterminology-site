# منصة التقييم المهني لمقدمي الخدمة

> حالة الحزمة: **نواة مستقلة غير مرتبطة بالإنتاج**. لا تُدمج أو تُنشر قبل مراجعة علمية وقانونية وترخيص الأدوات المطلوبة.

## الغرض

نظام دعم قرار مهني لمقدمي الخدمة والمراكز المتعاقدة والمعتمدين. يجمع بيانات متعددة المصادر والمجالات، ويفتح تقييمات لاحقة وفق قواعد موثقة، ويمنع إصدار حكم تشخيصي من أداة منفردة.

## مبادئ غير قابلة للتجاوز

1. لا تشخيص آلي، ولا استنتاج نهائي من مقياس واحد أو تقرير مصدر واحد.
2. كل أداة مرتبطة بإصدار محدد، ودليل، وترخيص، ولغة، وعينة تقنين، ومؤهل مستخدم.
3. لا تُعرض بنود أو مفاتيح تصحيح أو جداول معيارية محمية دون ترخيص كتابي صالح للرقمنة.
4. القرار النهائي للمختص أو الفريق المخول، مع اسم المراجع وتاريخ الاعتماد وسجل تدقيق.
5. نتيجة المسح تفتح مسارًا، ولا تثبت اضطرابًا.
6. النتائج المتعارضة أو غير الصالحة توقف المسار الآلي وتفرض مراجعة بشرية.
7. تراعي المنصة العمر الزمني والنمائي، اللغة، الثقافة، السمع، البصر، الحركة، التعليم والبيئة.
8. الملف النهائي يصف التشخيص عند اعتماده، والوظائف، والمشاركة، ونقاط القوة واحتياجات الدعم.

## النموذج التشغيلي

```text
إحالة وموافقة مستنيرة
  → فرز السلامة والمخاطر العاجلة
  → ملف نمائي/طبي/تعليمي/اجتماعي أولي
  → مسح متعدد المجالات
  → قواعد صلاحية وجودة البيانات
  → فتح المسارات المتخصصة
  → اختبارات تأكيدية وتشخيص تفريقي
  → معلومات من الأسرة والمدرسة والملاحظة المباشرة
  → مراجعة التعارض ونقص الأدلة
  → اجتماع/اعتماد الفريق
  → تقرير احتياجات ودعم ومتابعة
```

## مستويات النتيجة المسموح بها

- `insufficient_data`: بيانات غير كافية.
- `invalid_administration`: تطبيق غير صالح أو خارج شروط الدليل.
- `no_current_signal`: لا توجد إشارة كافية حاليًا.
- `monitor_and_rescreen`: متابعة وإعادة مسح في موعد موثق.
- `focused_assessment_required`: يلزم تقييم تخصصي محدد.
- `multidisciplinary_review_required`: يلزم فريق متعدد التخصصات.
- `urgent_referral`: إحالة عاجلة وفق بروتوكول السلامة.
- `clinician_validated_conclusion`: خلاصة اعتمدها مختص مخول؛ ليست ناتجًا آليًا.

## مكونات كل مسار

كل مسار حالة يجب أن يتضمن:

- تعريف نطاق المسار وما لا يغطيه.
- معايير الدخول والاستبعاد.
- إشارات الخطر والإحالة العاجلة.
- أدوات المسح المسموح بها.
- تقييمات الوظائف واللغة والمعرفة والسلوك التكيفي والحواس عند صلتها.
- التشخيصات والظروف الواجب استبعادها أو فحصها.
- قواعد الانتقال بين العقد، مع مصدر وإصدار وتاريخ مراجعة.
- شروط إيقاف المسار وإحالته للمراجعة البشرية.
- متطلبات الفريق المخول بالاعتماد.
- خطة المتابعة وإعادة القياس.

## الصلاحيات

| الدور | الصلاحيات الأساسية |
|---|---|
| منسق الإحالة | إنشاء الحالة، الموافقات، المواعيد، دون تفسير سريري |
| مقدم خدمة | إدخال الملاحظات والنتائج المصرح بها ضمن نطاقه |
| مختص أداة | تطبيق أداة محددة بعد التحقق من المؤهل والترخيص |
| قائد حالة | تنسيق البطارية ومراجعة الاكتمال والتعارض |
| مراجع سريري | اعتماد أو رفض الخلاصة وتوثيق السبب |
| مسؤول جودة | تدقيق السجلات والانحرافات دون تعديل النتيجة الأصلية |
| مسؤول نظام | إدارة الحسابات؛ لا يرى البيانات السريرية افتراضيًا |

## بوابات قبل تفعيل أي أداة

- إثبات حق الاستخدام والرقمنة.
- تحديد المؤهل المطلوب وتطبيق التحقق من الشهادات.
- مطابقة الإصدار واللغة والدليل.
- توثيق التقنين والصدق والثبات وحدود التفسير.
- اختبار دقة التصحيح مقابل حالات مرجعية مرخصة.
- اختبار الوصول، RTL، لوحة المفاتيح وقارئات الشاشة.
- اختبار حفظ التقدم والانقطاع والتزامن والتدقيق.
- موافقة اللجنة العلمية والقانونية وحماية البيانات.

## ضوابط محرك القواعد

- قواعد حتمية قابلة للتفسير والاختبار، لا أوامر مولدة لحظيًا.
- كل قاعدة تحمل: `source`, `version`, `effective_from`, `reviewed_by`, `test_cases`.
- لا تعمل قاعدة بعد انتهاء صلاحية مصدرها أو ترخيصها.
- لا تُحوَّل الدرجة الخام إلى معيارية إلا باستخدام جداول الإصدار واللغة المرخصة.
- يمنع المحرك الخلاصة عند غياب مصدر ثانٍ إذا كان البروتوكول يشترطه.
- يحتفظ النظام بالدرجة الخام، والتحويل، والتفسير، وأي تعديل يدوي كسجلات منفصلة.

## السلامة والخصوصية

- بيانات الأطفال والصحة بيانات شديدة الحساسية.
- تشفير أثناء النقل والتخزين، وفصل الهوية عن بيانات القياس عند الإمكان.
- أقل صلاحية لازمة، جلسات محدودة، مصادقة متعددة العوامل للمختصين.
- سجل غير قابل للمحو لفتح الملف والتطبيق والتعديل والتصدير والاعتماد.
- سياسة احتفاظ وحذف وتصدير وتصحيح حسب الدولة والعقود.
- لا استخدام للتدريب الآلي أو البحث إلا بموافقة مستقلة وآلية إخفاء هوية معتمدة.
- بروتوكول منفصل لحماية الطفل، وإيذاء النفس، والإساءة، والتدهور الطبي الحاد.

## المعايير المرجعية الأساسية

- WHO ICF لوصف الوظائف والإعاقة والمشاركة والعوامل البيئية.
- الإرشادات المهنية والتشخيصية الخاصة بكل حالة والدولة المستهدفة.
- Standards for Educational and Psychological Testing لمبادئ الصدق والثبات والعدالة والاستخدام.
- WCAG 2.2 AA كحد أدنى للإتاحة الرقمية.
- تقييم تنظيمي مستقل لوظائف دعم القرار قبل الإطلاق في كل سوق؛ بعض الوظائف قد تدخل ضمن البرمجيات الطبية.

## مراجع رسمية أولية

- WHO ICF: https://www.who.int/standards/classifications/international-classification-of-functioning-disability-and-health
- WHO-FIC: https://www.who.int/standards/classifications/icf
- W3C WCAG 2.2: https://www.w3.org/TR/WCAG22/
- FDA Clinical Decision Support Software, Final Guidance (2026): https://www.fda.gov/regulatory-information/search-fda-guidance-documents/clinical-decision-support-software

## مراحل التسليم

1. الحوكمة ونموذج البيانات والمحرك والقوالب.
2. مسار التأخر النمائي العام مع أدوات مسموحة قانونيًا.
3. مسارات التوحد، الإعاقة الذهنية، ADHD، اللغة وصعوبات التعلم.
4. الإعاقات السمعية والبصرية والحركية والمتعددة.
5. تقارير الفريق، مؤشرات الجودة، التدريب والاعتماد الداخلي.
6. دراسة تحقق واختبار ميداني قبل الربط بالإنتاج.

## تعريف الاكتمال

لا تُعد المنصة مكتملة أو معتمدة لمجرد اكتمال البرمجة. الاكتمال يتطلب لكل مسار: ترخيص الأدوات، مراجعة علمية مستقلة، تحقق تقني وسيكومتري، اختبار مستخدمين، موافقات حماية البيانات، تدريب المستخدمين، تجربة ميدانية، وخطة مراقبة وتصحيح بعد الإطلاق.
