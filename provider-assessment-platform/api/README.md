# عقد API لمنصة التقييم المهني

## الإصدار النشط

`openapi.v0.2.0.json`

هذا هو العقد الوحيد الذي يجوز استخدامه في بناء نموذج أولي أو إنشاء عميل أو خادم تجريبي. لا توجد خدمة منشورة، وجميع عناوين الخوادم ذات نطاق `.invalid` غير قابل للتوجيه.

## الإصدار التاريخي

`openapi.v0.1.0.json` محفوظ كسجل تصميم أولي فقط، وحالته **retired-draft**. يحتوي نموذج التقرير فيه على تعارض بنيوي اكتشف أثناء المراجعة، ولذلك:

- يمنع استخدامه في توليد شيفرة.
- يمنع اعتباره عقدًا صالحًا.
- لا يفحصه CI بوصفه الإصدار النشط.
- لا يحذف حتى يبقى سجل الخطأ والتصحيح قابلًا للتدقيق.

## مبادئ العقد النشط

- لا نقطة نهاية للتشخيص الآلي.
- لا نقطة نهاية للأهلية القانونية أو التعليمية الآلية.
- لا عرض لبنود الاختبارات أو التعليمات المحمية أو مفاتيح التصحيح أو جداول المعايير.
- تقييم المسار يعيد خطوة تالية قابلة للتفسير بثلاث حالات: `true` و`false` و`unknown`.
- بدء الجلسة يحتاج فحص وصول ناجحًا ومحدد المدة.
- إنشاء التقرير ينتج مسودة غير موقعة دائمًا.
- التوقيع عملية مستقلة تحتاج نطاق `report:sign` وتأكيد مراجعة وترخيصًا مهنيًا وبصمة محتوى.
- عمليات الكتابة الحساسة تحتاج مفتاح منع التكرار ومعرّف ارتباط وسجل تدقيق.
- تحديث الحالة يحتاج `If-Match` لمنع الكتابة فوق نسخة أحدث.

## سياسة الإصدارات

1. لا تعدل مواصفة نشطة مستخدمة في بيئة اختبار؛ ينشأ إصدار جديد.
2. كل إصدار يحمل حالة في هذا الملف: `active-draft` أو `retired-draft` أو `approved` أو `suspended`.
3. لا يصبح الإصدار `approved` قبل مراجعة أمن وخصوصية وتنظيم واختبارات تكامل فعلية.
4. يجب أن يتطابق إصدار العميل والخادم، ويمنع الخادم عمليات الكتابة عند عدم التطابق.

## الحالة الحالية

| الملف | الحالة | الاستخدام |
|---|---|---|
| `openapi.v0.2.0.json` | active-draft | التحقق والتصميم التجريبي فقط |
| `openapi.v0.1.0.json` | retired-draft | سجل تاريخي؛ ممنوع التنفيذ |
