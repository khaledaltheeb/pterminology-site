# منصة التقييم المهني المتدرّج — الإصدار 199

هذه وحدة مستقلة تمامًا عن التنقل العام وخرائط الموقع في مشروع «مصطلحات علم النفس / المنصة الشاملة للصحة النفسية وذوي الاحتياجات الخاصة».

## حالة الإصدار

- **الحالة:** ما قبل الاعتماد السريري والتنظيمي.
- **المستخدم المستهدف مستقبلًا:** المراكز ومقدمو الخدمة المتعاقدون والمعتمدون.
- **النشر العام:** معطل.
- **الفهرسة:** `noindex,nofollow,noarchive`.
- **الربط بالموقع الرئيسي:** غير منفذ عمدًا.
- **بيانات المرضى:** ممنوعة في هذه النسخة.
- **بنود المقاييس المحمية:** غير موجودة.
- **القرار التشخيصي الآلي:** محظور.

الوحدة الحالية هي نواة كاملة من ناحية **نموذج المعلومات، الحوكمة، مسارات الانتقال، دليل الأدوات، التدريب، ومحاكاة منطق الاستكمال**. ولا تصبح منصة تطبيق سريري فعلي إلا بعد استكمال تراخيص كل أداة، واعتماد النسخة العربية، وربط الهوية والصلاحيات والتشفير وسجل التدقيق، والمراجعة القانونية والتنظيمية.

## ما الذي تنفذه الوحدة؟

1. دليل مصنف للمقاييس والأدوات يوضح الغرض والعمر وطريقة التطبيق والمجيب والمؤهل والحقوق والدور والقيود والمقاييس المكملة.
2. ثلاثة عشر مسارًا متدرجًا للحالات والمجالات الأعلى أولوية:
   - اضطراب طيف التوحد.
   - الإعاقة الذهنية.
   - ADHD.
   - التأخر النمائي الشامل.
   - اضطراب اللغة النمائي.
   - اضطراب التعلم المحدد.
   - الشلل الدماغي.
   - الإعاقة السمعية.
   - الإعاقة البصرية.
   - اضطراب التناسق الحركي.
   - السلوكيات الصعبة وإيذاء الذات.
   - الإعاقات المتعددة والصمم الكففي.
   - الانتقال إلى الرشد والتأهيل المهني.
3. قواعد توقف عند الخطر أو نقص البيانات أو تعارض النتائج.
4. محاكاة تعليمية لا تستخدم بيانات حقيقية ولا تحسب درجات.
5. دورة تأسيسية من ستة دروس.
6. سبع بوابات جودة إلزامية قبل اعتماد أي تقرير.
7. فصل واضح بين المسح والتقييم والتشخيص والتصنيف وقياس التغير.
8. سجل مرجعي للمصادر المؤسسية.

## فلسفة القرار

لا تعتمد المنصة على قاعدة من نوع:

```text
درجة أكبر من X = تشخيص
```

بل على شبكة أدلة:

```text
سؤال إحالة
→ سلامة وصلاحية التطبيق
→ مسح عند الحاجة
→ تقييمات متعددة المجالات
→ أكثر من مصدر معلومات
→ تشخيصات تفريقية
→ مراجعة التعارض
→ حكم مختص مخول
→ خطة دعم ومتابعة
```

كل مرحلة في `catalog.json` تحتوي:

- الغرض.
- المجالات المطلوبة.
- الأدوات المرشحة.
- شرط الاكتمال.
- قواعد الانتقال.
- حالات التوقف أو الإحالة.

الأدوات المرشحة ليست أمرًا تلقائيًا بالتطبيق. يجب أن يختار المختص الأداة بعد التحقق من العمر واللغة والسياق والإعاقة الحسية والحركية والإصدار والترخيص.

## طريقة الاستخدام الحالية

### 1. تشغيل الوحدة

الوحدة ساكنة ولا تحتاج حزمة بناء:

```bash
python -m http.server 8080
```

ثم افتح:

```text
http://localhost:8080/professional-assessment-hub/
```

يجب عدم فتح `index.html` مباشرة عبر `file://` لأن تحميل `catalog.json` يحتاج خادم HTTP.

### 2. لوحة الحوكمة

ابدأ بقراءة القواعد غير القابلة للتجاوز. أي تطوير لاحق يجب أن يبقيها قابلة للاختبار آليًا، خصوصًا:

- منع التشخيص المنفرد.
- منع إدخال أداة دون حقوق موثقة.
- منع اعتماد تقرير دون مراجعة بشرية.
- منع استخدام نسخة أو ترجمة غير محددة.
- منع إخفاء النتيجة غير الصالحة أو المتعارضة.

### 3. المسارات السريرية

اختر الحالة واقرأ:

- شروط الدخول.
- إشارات الخطر.
- التصنيف المرجعي.
- المرحلة الأولى وشروطها.
- الأدوات المرشحة.
- شرط الانتقال.
- النتيجة التالية.

يجب ألا يبدأ الممارس من اسم اختبار. يبدأ من سؤال الإحالة، ثم يختار المسار، ثم يحدد المجال، ثم الأداة.

### 4. دليل المقاييس

استخدم البحث أو مرشح الحقوق. حالة الحقوق هي أهم حقل تشغيلي:

| الحالة | المعنى |
|---|---|
| `requires_publisher_license` | لا يجوز عرض البنود أو التصحيح أو التطبيق دون عقد وترخيص مناسب |
| `permission_required_before_embedding` | قد تكون الأداة متاحة بشكل ما، لكن التضمين أو الترجمة أو الاستخدام الإلكتروني يحتاج إذنًا |
| `classification_public_reference_only` | يمكن شرح النظام التصنيفي وإحالته، ولا يعني ذلك حرية تعديل النص أو العلامة |
| `internal_template` | نموذج داخلي غير معياري؛ لا يملك درجات معيارية أو تشخيصية |
| `requires_professional_service` | خدمة يجب أن يؤديها اختصاصي خارج محرك الاستبيانات |
| `manual_rights_review` | لا ينفذ قبل مراجعة المصدر والحقوق والنسخة والترجمة يدويًا |

القيمة الافتراضية لكل أداة معيارية هي:

```text
blocked_until_rights_and_local_validation
```

ولا تتغير إلى `approved_for_production` إلا بقرار موثق.

### 5. محاكاة الحالة

المحاكاة تقبل فقط:

- مسارًا.
- عمرًا.
- عدد مصادر المعلومات.
- حالة السمع.
- حالة البصر.
- وجود خطر عاجل.

لا تقبل اسمًا أو رقمًا أو تاريخ ميلاد أو تشخيصًا أو نصًا حرًا. هذا مقصود لمنع استخدام النموذج التجريبي كملف مريض.

المنطق:

1. إذا وجد خطر عاجل، يوقف المسار.
2. إذا كان المصدر واحدًا، يطلب مصدرًا ثانيًا.
3. إذا لم يوثق السمع أو البصر، يمنع التفسير الكامل.
4. إذا اكتملت البوابات الأولية، يعرض مراحل المسار فقط.
5. لا يحسب درجة، ولا يختار نسخة اختبار، ولا يصدر تشخيصًا.

## دليل التطبيق المهني المستقبلي

### قبل فتح الحالة

يجب أن يتحقق النظام من:

- هوية المركز واتفاقية الاستخدام.
- حالة اعتماد الممارس.
- المهنة ونطاق الممارسة.
- التدريب الخاص بالمقياس.
- تاريخ انتهاء الترخيص.
- اللغة التي يتقنها المطبق.
- موافقة الشخص أو ولي الأمر.
- أساس قانوني لمعالجة البيانات.
- بروتوكول الطوارئ في بلد المركز.
- إمكانية الوصول ووسيلة التواصل.

### قبل اختيار المقياس

يسجل الممارس:

1. سؤال الإحالة.
2. العمر الزمني والعمر المصحح إن كان مناسبًا.
3. اللغة أو اللغات واللهجات المستخدمة.
4. طريقة التواصل.
5. حالة السمع والبصر.
6. القيود الحركية أو الحسية.
7. التعليم والغياب وفرص التعلم.
8. الأدوية والحالة الطبية ذات الصلة.
9. مصادر المعلومات المتاحة.
10. سبب اختيار الأداة بدل البدائل.

### قبل التطبيق

يتحقق النظام من:

- اسم المقياس وإصداره.
- لغة النسخة.
- مالك الحقوق.
- رقم اتفاقية الترخيص.
- المعايير المستخدمة.
- العمر المقبول.
- المؤهل المطلوب.
- التدريب المطلوب.
- طريقة التطبيق المسموحة: حضوري أو عن بعد أو ورقي أو رقمي.
- السماح بإعادة الاختبار والفاصل الزمني.
- قواعد البدء والتوقف والتلقين.
- إعدادات الوصول المسموح بها.

### أثناء التطبيق

يجب توثيق:

- تاريخ ووقت ومكان الجلسة.
- المطبق والحاضرين.
- حالة الشخص والتعب والدافعية.
- أي انقطاع.
- أي تعديل أو مساعدة.
- الأجهزة المستخدمة.
- الأخطاء التقنية.
- العناصر غير المكتملة.
- سبب التوقف.

المواد السرية لا تحفظ في سجل الحالة العام. تحفظ في مخزن منفصل مشفر ومقيد، أو لا تحفظ إذا كان عقد الناشر يمنع ذلك.

### بعد التطبيق

1. تحقق مزدوج من الإدخال والتصحيح.
2. تثبيت الإصدار والمعيار.
3. تحديد صلاحية النتيجة: صالحة، محدودة، غير صالحة.
4. مقارنة المصادر دون أخذ متوسط آلي للتعارض.
5. تشغيل قواعد الانتقال.
6. طلب المجالات الناقصة.
7. إظهار التشخيصات التفريقية.
8. إنشاء مسودة تقرير.
9. مراجعة بشرية.
10. توقيع واعتماد وتجميد نسخة التقرير.
11. أي تعديل لاحق ينشئ نسخة جديدة ولا يمحو السابقة.

## الحد الأدنى للتقرير

- بيانات تعريف محدودة وفق الحاجة.
- سؤال الإحالة.
- الموافقة ومصدرها.
- من شارك في التقييم.
- مصادر المعلومات.
- الأدوات والإصدارات واللغة والمعايير.
- مؤهلات المطبق.
- ظروف التطبيق والتعديلات.
- النتائج والدرجات المسموح بعرضها.
- صلاحية كل نتيجة.
- تفسير متعدد المجالات.
- اختلاف المصادر.
- التشخيصات التفريقية.
- المخاطر.
- نقاط القوة.
- الاحتياجات الوظيفية.
- توصيات محددة وقابلة للتنفيذ.
- التقييمات التالية وسببها وأولويتها.
- حدود التقرير.
- اسم وتوقيع المختص.
- المراجع الثاني عند الحاجة.
- رقم نسخة التقرير وسجل التغيير.

## قواعد خاصة بالتعارض

لا يجوز أن يقوم المحرك بـ:

- حساب متوسط بين تقييم الوالد والمعلم ثم إخفاء الاختلاف.
- اعتبار اختبار محوسب بديلًا عن تقرير البيئات.
- تعويض غياب السلوك التكيفي بدرجة ذكاء.
- تعويض غياب فحص السمع باختبار لغة.
- اعتبار الملاحظة القصيرة وصفًا لجميع البيئات.
- تفسير عدم الاستجابة لدى شخص ذي إعاقة متعددة كغياب قدرة.
- استخدام نسخة مترجمة غير مقننة كدرجة معيارية محلية.

عند التعارض يجب أن يطلب النظام:

- مدة معرفة كل مصدر بالشخص.
- أمثلة قابلة للملاحظة.
- اختلاف متطلبات البيئات.
- توقيت الدواء أو النوم أو الضغط.
- مصدرًا ثالثًا أو ملاحظة مباشرة.
- فحص صلاحية الترجمة أو طريقة الوصول.
- اجتماع مراجعة حالة.

## الدورة التدريبية

الدورة داخل الواجهة تتكون من:

1. الفرق بين المسح والتقييم والتشخيص.
2. اختيار الأداة وصلاحية التطبيق.
3. التطبيق المعياري وأمن الاختبار.
4. التفسير متعدد المصادر والتعارض.
5. كتابة التقرير وخطة الإحالة.
6. الحوكمة والخصوصية وضبط الجودة.

هذه الدورة تعريفية فقط. لا تمنح تدريب ADOS أو اختبارات Wechsler أو أي تأهيل ناشر، ولا تستبدل الإشراف السريري.

## المتطلبات قبل الربط بالموقع

### حقوق المقاييس

يجب إنشاء سجل مستقل لكل مقياس يضم:

- صاحب الحقوق.
- جهة الاتصال.
- العقد.
- الدول المشمولة.
- اللغات.
- السماح بالرقمنة.
- السماح بالتخزين السحابي.
- السماح بعرض البنود.
- السماح بالتصحيح الآلي.
- عدد الاستخدامات.
- تاريخ الانتهاء.
- قيود التدريب.
- متطلبات حذف البيانات.
- إجراءات إنهاء العقد.

### التقنين العربي

لا تعتمد أي نسخة عربية قبل:

- إذن الترجمة.
- ترجمة أمامية وعكسية.
- لجنة خبراء.
- مقابلات معرفية.
- دراسة تجريبية.
- ثبات وصدق.
- حساسية ونوعية عندما يكون للمقياس غرض مسحي.
- تكافؤ القياس وتحليل تحيز البنود.
- معايير مرتبطة بالسكان عند الحاجة.
- دليل تقني منشور.
- تحديد البلدان واللهجات التي تغطيها.

### الأمن والخصوصية

قبل البيانات الحقيقية يلزم:

- تسجيل دخول مؤسسي ومصادقة متعددة العوامل.
- صلاحيات دقيقة حسب الدور والمركز والحالة.
- تشفير أثناء النقل والتخزين.
- فصل الهوية عن بيانات القياس.
- سجل تدقيق غير قابل للتعديل من المستخدم.
- إدارة موافقات.
- سياسات احتفاظ وحذف.
- نسخ احتياطي واختبار استعادة.
- منع التحليلات الإعلانية.
- منع تدريب نماذج الذكاء الاصطناعي على بيانات الحالات افتراضيًا.
- خطة استجابة للحوادث.
- تقييم أثر حماية البيانات.
- مراجعة القوانين في الأردن وكل بلد تشغيل.

### الإتاحة

الهدف الأدنى WCAG 2.2 AA، مع اختبار بشري يشمل:

- قارئ شاشة عربي.
- لوحة المفاتيح فقط.
- تكبير 200% و400%.
- التباين.
- ترتيب التركيز.
- عدم الاعتماد على اللون.
- رسائل أخطاء واضحة.
- عدم طلب إدخال متكرر دون داع.
- وسائل مصادقة قابلة للوصول.
- لغة مبسطة عند الحاجة.
- وقت إضافي وحفظ تقدم.
- دعم التواصل المعزز والبديل.

## مبدأ الدمج المستقبلي

عند اكتمال الاعتمادات فقط:

1. بناء خدمة هوية وصلاحيات مستقلة.
2. نقل `catalog.json` إلى سجل نسخ محكوم.
3. إضافة قاعدة بيانات مشفرة وسجل تدقيق.
4. إدخال أداة واحدة مرخصة في كل مرة.
5. اختبارها على بيئة منفصلة.
6. مراجعة علمية وقانونية وأمنية.
7. تجربة محدودة مع مراكز مختارة.
8. قياس الأخطاء والتعارضات.
9. ربط بوابة الممارسين دون إدخالها في صفحات الجمهور.
10. عدم إضافة URL إلى Sitemap العام إلا إذا كان محتواه عامًا وغير سري.

## المصادر المرجعية الأساسية

توجد الروابط داخل `catalog.json`. وجود المصدر لا يعني أن المنصة معتمدة من الجهة ولا يمنح حق استخدام اختبار تجاري. يجب عدم استخدام شعارات الجهات أو عبارات مثل «معتمد من» دون اتفاق مكتوب.
