# عقد تدقيق الروابط بين أقسام الموقع v177

## الغرض

الموقع يحتوي موسوعة ومدونة وأدلة ومقاييس ومركزًا للأسرة وذوي الاحتياجات الخاصة. وجود الصفحة في خريطة الموقع لا يكفي؛ يجب أن يتمكن المستخدم ومحرك البحث من اكتشافها من صفحات مرتبطة بالموضوع والجمهور.

## ما يفحصه المدقق

يفحص كل ملف HTML في حزمة الموقع المبنية ويصدر قرارًا مستقلًا لكل URL:

- عدد الروابط الداخلية الواردة.
- الأقسام المختلفة التي تقود إلى الصفحة.
- الصفحات اليتيمة أو ضعيفة الاكتشاف.
- الروابط الداخلية المكسورة.
- العنوان والوصف وH1 وCanonical وJSON-LD.
- القسم الذي تنتمي إليه الصفحة: الموسوعة، المدونة، ذوو الاحتياجات، الأدلة، الاختبارات، الأدوات المعرفية، الأسرة أو الثقة.

## القرارات

- `pass`: لا توجد فجوة مكتشفة في العقد.
- `improve`: الصفحة موجودة لكنها ضعيفة الاكتشاف أو الميتاداتا وتحتاج تحسينًا تحريريًا أو روابط سياقية.
- `fix-before-publish`: عيب حرج مثل رابط داخلي مكسور أو Canonical مفقود أو عدد H1 غير صالح.

## قواعد الروابط

لا تعالج النتائج بإضافة قائمة روابط عامة متكررة إلى جميع الصفحات. الإصلاح المقبول يجب أن يكون سياقيًا، مثل:

- ربط مصطلح في الموسوعة بمقال يشرح الفرق أو الاستخدام.
- ربط مقال بدليل عملي أو أداة متابعة مناسبة.
- ربط صفحة طفل أو أسرة بمركز ذوي الاحتياجات فقط عندما توجد صلة حقيقية.
- ربط دليل بصفحة الثقة أو المصادر عند شرح الحدود المهنية.

## حدود المدقق

- لا يثبت صحة المحتوى العلمي.
- لا يقرر التشخيص أو العلاج.
- لا يضيف روابط آليًا ولا يعدل الصفحات.
- لا يضع `noindex` ولا يحذف صفحة تلقائيًا.
- النتائج قائمة عمل للمراجعة البشرية والناشرين المستقلين.

## الاستخدام

```bash
python scripts/audit_cross_section_links_v177.py _site \
  --json artifacts/cross-section-links-v177.json \
  --csv artifacts/cross-section-links-v177.csv
```

يجب تشغيله على حزمة الإنتاج الموثقة نفسها قبل استخدام الأرقام لاتخاذ قرارات تحريرية.
