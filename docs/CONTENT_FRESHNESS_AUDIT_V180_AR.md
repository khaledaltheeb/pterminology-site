# تدقيق حداثة المحتوى والمراجعة v180

يفحص هذا المدقق سجلات المحتوى في `content/` و`data/` وصفحات HTML المبنية عند توفر `_site/`. هدفه كشف الصفحات التي تبدو سليمة تقنيًا لكنها تفتقد تاريخ مراجعة أو حالة مراجعة واضحة أو مصادر منظمة حديثة.

## القرارات

- `current`: لا توجد فجوة مكتشفة وفق العقد.
- `needs-update`: يحتاج السجل تحديث مراجعة أو مصدر، دون حذف تلقائي.
- `fix-before-publish`: نقص جوهري يمنع اعتبار السجل جاهزًا، خصوصًا للمحتوى الصحي مرتفع المخاطر.

## ما يفحصه

حالة النشر، مستوى المخاطر، تاريخ آخر مراجعة، حالة المراجعة، وجود المصادر المنظمة وتاريخ تحققها، وعناصر HTML الأساسية: العنوان، وصف الميتا، Canonical و`dateModified`.

## حدود صريحة

لا يثبت المدقق صحة المعلومة الطبية أو صفة المراجع، ولا يغيّر حالة النشر، ولا يحذف أو يضع `noindex`، ولا يثبت الظهور الحي. تقارير JSON وCSV هي قائمة عمل تحريرية فقط.

## التشغيل

```bash
python scripts/audit_content_freshness_v180.py . --today 2026-07-23
```

الإعداد الافتراضي: مراجعة المحتوى خلال 365 يومًا، والتحقق من المصدر خلال 730 يومًا. النسخة v180 تصلح خطأ قراءة صفحات `_site` الذي ظهر في أول دورة اختبار، وتحتفظ بالمسار النسبي في التقرير مع قراءة الملف من مساره الفعلي.
