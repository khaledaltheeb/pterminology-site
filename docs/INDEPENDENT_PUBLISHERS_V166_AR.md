# نظام الناشرين المستقلين v166

## القاعدة الأساسية

كل ناشر مسؤول عن قسم واحد فقط، ويملك دورة العمل كاملة من اختيار الموضوع إلى التحقق من ظهوره على النسخة الحية. لا يوجد ناشر يكتب جزءًا ثم يترك ناشرًا آخر يكمله، ولا يوجد وكيل SEO منفصل يضاف في النهاية؛ SEO والمصادر والسلامة والروابط والاختبارات جزء من مسؤولية الناشر نفسه.

## معنى الاستقلال

لكل ناشر مجلد تحت `content/publishers/`، ومسار URL، وأنواع محتوى مسموحة، وQueue، وأولويات، ومصادر، ومعايير قبول. يمنع المدقق تكرار المعرفات والـslugs ومسارات URL، ويمنع وضع مقال داخل ناشر القاموس أو اختبار داخل ناشر المرأة مثلًا.

## دورة النشر

1. اختيار موضوع ذي حاجة وقيمة وفجوة حقيقية.
2. تحديد نية البحث والجمهور والعمر والمخاطر.
3. جمع المصادر الأولية وتسجيل معرفاتها.
4. كتابة مخطط يمنع الحشو والتكرار.
5. كتابة محتوى عربي أصلي غني ومتخصص.
6. مراجعة الدقة والسلامة والحقوق واللغة.
7. إعداد العنوان والوصف وcanonical وOpen Graph وTwitter Card وBreadcrumb وJSON-LD.
8. إضافة روابط سياقية داخلة وخارجة.
9. تشغيل اختبارات المحتوى والبناء والإتاحة والمتصفح حسب نوع الصفحة.
10. بناء artifact موثقة ودمج PR.
11. نشر GitHub Pages والتحقق من SHA والملفات الحية.
12. تغيير حالة العنصر إلى `published` وتسجيل تاريخ المراجعة والإصدار.

## الاستمرارية والتردد

النظام يعمل كطابور مستمر: بعد ثبوت نشر العنصر السابق حيًا، ينتقل الناشر إلى العنصر التالي. الحد الأقصى PR نشر نشط واحد لكل ناشر لتجنب التضارب، ويمكن أن يعمل الناشرون العشرة بالتوازي. لا تعني السرعة السماح بالصفحات الرقيقة أو تجاوز التحقق.

## جودة المحتوى

لا يقبل العنصر إذا كان مجرد تعريف قصير أو إعادة صياغة لمحتوى موجود. يجب أن يكون كاملًا بالنسبة لنيته: تعريف أو جواب مباشر، فروق، سياق، خطوات عملية، حدود، أسئلة أو أخطاء شائعة عند صلتها، مصادر، ومتى تطلب المساعدة في الموضوعات الحساسة.

## حدود الأتمتة

GitHub Actions تستطيع التحقق والبناء والنشر المجدول عند وجود محتوى جاهز، لكنها لا تثبت وحدها صحة المعلومة أو توفر مراجعة اختصاصي. لا توصف أي صفحة بأنها منشورة حتى ينجح النشر وتثبت مطابقة SHA على الموقع الحي.
