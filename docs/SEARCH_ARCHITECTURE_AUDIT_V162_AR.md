# تدقيق SEO وبنية البحث الداخلي v162

## الهدف

يفصل هذا التدقيق بين جودة النص العلمي وبين قابلية اكتشافه وفهمه. فهو يبني رسمًا للصفحات والروابط وخرائط الموقع، ويكشف التعارضات التي قد تجعل صفحة جيدة غير قابلة للفهرسة أو معزولة عن رحلة المستخدم.

## ما يفحصه

- وجود عنوان ووصف وH1 واحد وCanonical.
- أن Canonical يستخدم HTTPS والنطاق والمسار الرسميين.
- تكرار Canonical بين أكثر من صفحة.
- تعارض `noindex` مع إدراج الصفحة في Sitemap.
- قراءة Sitemap Index والخرائط الفرعية والتحقق من الملفات المشار إليها.
- وجود الصفحات القابلة للفهرسة داخل خرائط الموقع.
- الروابط الداخلية الواردة والصادرة والصفحات اليتيمة.
- الروابط الداخلية المكسورة داخل artifact.
- العناوين والأوصاف المكررة.
- JSON-LD وأنواع Schema.
- نطاق روابط hreflang.
- النصوص العامة أو الفارغة للروابط مثل «هنا» و«اقرأ المزيد».

## مستوى الأخطاء

تُعامل المشاكل البنيوية الخطرة كأخطاء، ومنها Canonical مفقود أو خارج النطاق، JSON-LD غير صالح، Canonical مكرر، `noindex` داخل Sitemap، أو خريطة موقع مفقودة/فاسدة.

أما الصفحات اليتيمة، نقص Schema، العنوان المكرر، غياب الصفحة عن Sitemap، والنص العام للرابط فهي إشارات تحسين. لا يُحذف محتوى أو يتغير Canonical تلقائيًا بسببها.

## طريقة التشغيل

بعد الدمج، يعمل Workflow مستقل فقط بعد نجاح حزمة الإنتاج الموثقة على `main`. ينزل artifact من run-id نفسه، ويشغل المدقق عليها، ويرفع JSON وCSV وملف provenance. لا يكتب إلى المستودع ولا يعيد النشر.

## المخرجات

- `search-architecture-v162.json`: الملخص والأخطاء والتحذيرات وسجل كل صفحة.
- `search-architecture-v162.csv`: جدول قابل للفرز يضم الفهرسة وCanonical والروابط وSchema.
- `provenance.json`: SHA وتشغيل الإنتاج الذي جاءت منه الحزمة.

## الحدود

التدقيق لا يقدّر حجم البحث، ولا يثبت جودة المحتوى الطبي، ولا يقرر دمج الصفحات أو تحويلها، ولا يضمن فهرسة محركات البحث. بيانات Search Console تظل طبقة منفصلة عند ربط الحساب الصحيح.
