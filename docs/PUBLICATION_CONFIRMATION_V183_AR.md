# تأكيد النشر بعد كل عمل مكتمل

## القاعدة

لا يُعد أي عمل مدمج في `main` منشورًا أو مكتملًا تشغيليًا لمجرد نجاح الدمج أو بناء Artifact. تكتمل الحالة فقط عندما:

1. تنجح بوابة الإنتاج الشاملة على `main`.
2. تُنزّل حزمة `validated-production-site` الخاصة بنفس تشغيل البوابة.
3. يطابق `deployment.json` رقم SHA الكامل للـcommit الذي اجتاز الاختبارات.
4. تُنشر الحزمة نفسها على GitHub Pages.
5. يظهر `deployment.json` الحي بنفس SHA.
6. يطابق `release-manifest.json` الحي سجل الحزمة المنشورة.
7. تُفحص كل الملفات الجديدة أو المتغيرة بالبصمة والحجم على الموقع الحي.
8. يُتأكد من اختفاء الملفات المحذوفة من الموقع الحي.
9. يُحفظ تقرير إثبات باسم `live-publication-confirmation-v183-<SHA>`.

## الحالة الوحيدة المسموح بها بعد النجاح

`published-and-live-confirmed`

أي فشل في SHA أو البصمة أو الحجم أو اختفاء ملف جديد أو بقاء ملف محذوف يجعل Workflow يفشل، ولا يجوز وصف العمل بأنه منشور.

## عدم تغيير بقية سير العمل

هذه البوابة تعمل بعد نجاح بوابة الإنتاج القائمة، ولا تغيّر الناشرين أو المحتوى أو اختبارات الجودة الحالية. وظيفتها الوحيدة هي تحويل النشر والتأكد الحي إلى خطوة إلزامية بعد كل عمل مكتمل على `main`.
