# سياسة الثقة والسلامة المؤسسية

**المنصة:** مصطلحات علم النفس  
**الإصدار:** 1.0  
**تاريخ الإصدار:** 2026-07-21  
**حالة الوثيقة:** سياسة تشغيلية أولية قابلة للتحديث

## 1. الغرض والنطاق
تهدف هذه السياسة إلى توضيح كيفية إعداد المحتوى النفسي والصحي ومراجعته وتحديثه، وكيفية التعامل مع الأدوات والاختبارات والبيانات المحلية والتصحيحات وحالات الخطر. تنطبق على صفحات الموسوعة والأدلة والمقالات والمسارات التعليمية والأدوات والاختبارات والمواد القابلة للطباعة.

المنصة تعليمية وتوعوية. لا تقدم تشخيصًا طبيًا أو نفسيًا، ولا تستبدل الطبيب أو الأخصائي، ولا تقدم وصفات دوائية أو خطط علاج شخصية.

## 2. هوية الجهة المالكة والفريق
تُعرض فقط الأسماء والصفات المهنية والشراكات التي يمكن توثيقها علنًا أو بمستندات داخلية قابلة للمراجعة. لا يجوز إنشاء أسماء خبراء أو لجان علمية أو اعتمادات أو شراكات أو أرقام أثر غير حقيقية.

عند عدم وجود مراجعة اختصاصي موثقة، يجب استخدام وصف صريح مثل: **«محتوى تحريري مبني على مصادر منشورة ولم يخضع بعد لمراجعة اختصاصي موثق»**.

## 3. منهجية التحرير
يمر المحتوى، بحسب حساسيته، بالمراحل التالية:

1. تحديد السؤال والجمهور وحدود الصفحة.
2. جمع مصادر أولية وإرشادات مهنية حديثة.
3. صياغة عربية أصلية دون نسخ مطول أو إعادة صياغة قريبة من مصدر محمي.
4. التحقق من الادعاءات المهمة ومطابقتها للمصادر.
5. فحص اللغة التشخيصية والوعود العلاجية والوصم.
6. مراجعة الاتساق والوضوح والروابط والمراجع.
7. تسجيل تاريخ المراجعة وحالتها.

يجب فصل الحقائق المثبتة عن الآراء والتجارب الشخصية والافتراضات. لا تُستخدم عناوين مثيرة أو وعود بالشفاء أو التخويف لتضخيم التفاعل.

## 4. سياسة المصادر
ترتيب الأفضلية:

1. الإرشادات الرسمية والهيئات الصحية والمهنية.
2. المراجعات المنهجية والتحليلات التلوية.
3. الدراسات المحكمة ذات المنهج الواضح.
4. الكتب الأكاديمية والمراكز الجامعية الموثوقة.
5. المصادر التثقيفية الثانوية عند الحاجة إلى التبسيط فقط.

لا يُعتمد على منشورات التواصل الاجتماعي أو المدونات التجارية أو المحتوى المجهول لإثبات ادعاء طبي أو نفسي مهم. يجب توثيق الادعاءات الحساسة بمصدر مباشر كلما أمكن.

## 5. تصنيف قوة الدليل
تستخدم المنصة التصنيف التشغيلي التالي:

- **قوي:** إرشادات مهنية متوافقة أو مراجعات منهجية عالية الجودة.
- **متوسط:** دراسات متعددة متسقة مع بعض القيود.
- **محدود:** دراسات قليلة أو صغيرة أو غير متسقة.
- **غير مؤكد:** لا تتوفر أدلة كافية أو توجد خلافات جوهرية.

هذا التصنيف لا يعني حكمًا سريريًا نهائيًا، ويجب شرح حدود الدليل عند تأثيرها في القرار.

## 6. المراجعة العلمية وتاريخ التحديث
لكل مادة حساسة يجب إظهار أو تخزين:

- تاريخ آخر مراجعة.
- حالة المراجعة: مسودة، راجعت تحريريًا، تحتاج مراجعة اختصاصي، أو راجعها اختصاصي موثق.
- المصادر الأساسية.
- سبب التحديث الجوهري إن وجد.

لا يجوز وصف المادة بأنها «مراجعة علميًا» إلا عند وجود مراجعة فعلية يمكن توثيقها.

## 7. إخلاء المسؤولية وحدود الاستخدام
المعلومات عامة ولا تناسب كل شخص. التشخيص والعلاج والدواء تتطلب تقييمًا مهنيًا. لا ينبغي بدء دواء أو إيقافه أو تغيير جرعته بناءً على محتوى المنصة.

عند وجود أعراض جديدة أو شديدة أو تدهور مفاجئ أو آثار جانبية أو خطر على النفس أو الآخرين، يجب التواصل مع جهة صحية مؤهلة أو خدمات الطوارئ المحلية.

## 8. سلامة المحتوى النفسي ومنع التشخيص الذاتي
يجب أن:

- تشرح الصفحات الفرق بين التوعية والتشخيص.
- تعرض الأعراض كإشارات محتملة لا كحكم نهائي.
- تذكر الحالات المتشابهة والأسباب الطبية أو البيئية المحتملة عند الصلة.
- تتجنب لغة الوصم أو لوم المريض والأسرة.
- تمنع الوعود بالشفاء والعلاجات غير المثبتة.
- توضح متى تكون المتابعة المهنية ضرورية.

## 9. الأدوات والاختبارات والمسارات التعليمية
كل أداة أو اختبار يجب أن يوضح قبل البدء وبعد النتيجة:

- الغرض والفئة المستهدفة.
- هل هو تعليمي أو للمتابعة أو للفرز غير التشخيصي.
- طريقة الحساب وحدود النتيجة.
- أن النتيجة لا تثبت اضطرابًا ولا تنفيه.
- ما الذي يمكن فعله بعد النتيجة.
- متى يلزم طلب مساعدة مهنية أو عاجلة.
- مكان تخزين البيانات وكيفية حذفها.

لا يجوز عرض درجات سريرية أو معايير تشخيصية على أنها معتمدة ما لم تكن الأداة مرخصة ومطبقة وفق شروطها.

## 10. الخصوصية والبيانات المحلية
المبدأ الافتراضي هو تقليل البيانات. لا تُجمع بيانات حساسة أو تُرسل إلى خادم دون حاجة واضحة وموافقة موثقة.

عند استخدام التخزين المحلي في المتصفح:

- يجب إبلاغ المستخدم بوضوح.
- يجب توفير حذف شامل وسهل للبيانات المحلية.
- يجب تجنب تخزين أسماء كاملة أو معلومات طبية تعريفية ما لم توجد ضرورة معلنة.
- يجب ألا تُستخدم البيانات المحلية للإعلانات أو التتبع الخفي.

لا تعني هذه الوثيقة أن المنصة تستخدم ملفات ارتباط أو تحليلات أو خوادم حاليًا؛ يجب توثيق كل خدمة فعلية عند إضافتها.

## 11. ملفات الارتباط والتحليلات
عند إضافة ملفات ارتباط أو أدوات تحليل:

- تصنف إلى ضرورية، تفضيلات، تحليل، أو تسويق.
- لا تُفعّل غير الضرورية قبل الموافقة عندما يتطلب القانون ذلك.
- يوضح المزود والغرض ومدة الاحتفاظ وطريقة الرفض.
- تمنع مشاركة بيانات الصحة النفسية لأغراض إعلانية موجهة.

## 12. التواصل والشكاوى
يجب نشر وسيلة تواصل صحيحة يمكن استخدامها لطلب تصحيح أو الإبلاغ عن محتوى ضار أو مشكلة خصوصية. لا يجوز عرض بريد أو هاتف غير مراقب على أنه قناة دعم عاجل.

## 13. سياسة التصحيح وسجل التحديثات
عند ثبوت خطأ:

1. تقييم مستوى الضرر.
2. تصحيح الخطأ بسرعة مناسبة لحساسيته.
3. توثيق التغيير الجوهري وتاريخه.
4. توضيح الخطأ السابق عند احتمال تأثيره في قرارات المستخدمين.
5. مراجعة الصفحات المرتبطة لمنع تكراره.

التعديلات الإملائية البسيطة لا تتطلب سجلًا علنيًا، أما التغييرات الطبية أو النفسية الجوهرية فتتطلب سجلًا واضحًا.

## 14. تضارب المصالح والشراكات
يجب الإفصاح عن التمويل والرعاية والروابط التابعة والعلاقات التجارية التي قد تؤثر في المحتوى. لا يجوز للممول أو الشريك فرض نتيجة علمية أو إخفاء دليل مخالف.

لا تُعرض أي شراكة أو اعتماد إلا بعد توثيقها. الروابط التابعة، إن وجدت، يجب أن تكون معلنة وألا تغيّر ترتيب التوصيات أو صياغة الأدلة.

## 15. الأزمات والمساعدة العاجلة
الموقع ليس خدمة طوارئ ولا يراقب المستخدمين لحظيًا. عند وجود خطر مباشر أو وشيك، أو أفكار انتحارية مع نية أو خطة، أو تهديد للآخرين، أو فقدان شديد للسيطرة، يجب توجيه المستخدم إلى خدمات الطوارئ المحلية أو أقرب قسم طوارئ أو شخص موثوق قادر على البقاء معه.

لا تُعرض أرقام طوارئ حسب الدولة دون تحقق دوري من صحتها. وعند غياب قائمة محلية موثقة، تستخدم صياغة عامة واضحة بدل أرقام قديمة أو غير دقيقة.

## 16. الشفافية أمام الجهات المانحة والجمهور
يجب التمييز بين:

- ما خُطط له.
- ما نُفذ واختبر.
- ما نُشر فعليًا.
- ما تمت مراجعته علميًا.
- ما لا يزال تجريبيًا أو غير مؤكد.

لا تُستخدم أعداد الصفحات أو الزيارات أو المستفيدين أو الادعاءات العالمية دون قياس قابل للتحقق وتاريخ محدد.

## 17. الحوكمة والمراجعة الدورية
تراجع هذه السياسة عند إضافة نوع بيانات جديد، أو أداة حساسة، أو شريك، أو خدمة تحليل، أو عند ظهور خطأ مؤثر. الحد الأدنى المقترح للمراجعة الدورية مرة كل 12 شهرًا، مع مراجعة عاجلة عند تغير جوهري.
