# عقد متطلبات الثقة والسلامة

آخر مراجعة: 22 يوليو 2026

يحوّل هذا العقد متطلبات الثقة والسلامة من توجيهات عامة إلى شروط قبول يمكن فحصها آليًا للأدوات والاختبارات والمسارات التعليمية والأدلة ومسارات التعافي. وهو لا يدّعي اعتمادًا طبيًا أو قانونيًا، ولا يحل محل المراجعة العلمية المتخصصة عندما تكون مطلوبة.

## لماذا يوجد هذا العقد؟

قد تبدو أداة أو صفحة مكتملة بصريًا بينما لا تشرح حدودها، أو ما يحدث للبيانات، أو ما ينبغي فعله بعد النتيجة. لذلك يفرض العقد حقولًا واضحة بحسب نوع المحتوى ويمنع نشر ادعاءات تشخيصية أو شراكات أو مراجعات غير موثقة.

## المتطلبات المشتركة

كل مادة مشمولة يجب أن توضح، بحسب نوعها:

- الغرض والجمهور المقصود.
- الحدود وما لا تستطيع الأداة أو المادة إثباته.
- تنبيهًا صريحًا بأنها ليست تشخيصًا ذاتيًا.
- مصدر المعلومة أو أساس الحساب عندما توجد نتيجة.
- طريقة التحقق من المدخلات وحدود الدرجات.
- الخصوصية، ومكان التخزين، ومدة الاحتفاظ، وطريقة سحب الموافقة والتصدير والحذف الكامل.
- تأكيدًا واضحًا بعد نجاح الحذف أو رسالة خطأ قابلة للوصول عند تعذره.
- الخطوة التالية بعد النتيجة أو بعد إكمال المسار.
- تاريخ المراجعة وحالتها ومستوى السلامة.
- مسارًا لطلب المساعدة المهنية عند الحاجة في المحتوى الحساس.
- مسار مساعدة عاجلة محلي فقط عند تناول أزمة أو خطر مباشر أو وجود معيار تصعيد عاجل صريح.

## مستويات السلامة ومسارات المساعدة

يفصل العقد بين المساعدة المهنية غير العاجلة والطوارئ حتى لا تُعرض رسائل أزمة غير متناسبة داخل أدوات متابعة منخفضة الخطورة.

- **عام:** لا يتعامل مع نتيجة فردية حساسة، ولا يفرض مسار مساعدة مهنية أو طوارئ.
- **حساس:** يتعامل مع أعراض أو نتيجة فردية أو بيانات نفسية محلية؛ يجب أن يشرح متى وكيف يطلب المستخدم مساعدة مهنية، لكنه لا يفرض مسار طوارئ افتراضيًا.
- **عاجل:** يتناول خطرًا مباشرًا أو إيذاء الذات أو الآخرين أو أزمة نفسية أو معيار تصعيد عاجل صريح؛ يجب أن يتضمن مسار مساعدة مهنية ومسار طوارئ محليًا مناسبًا.

لا يجوز ترقية محتوى إلى المستوى العاجل لمجرد أنه نفسي، ولا يجوز خفضه عندما يتضمن خطرًا مباشرًا أو معيار تصعيد واضحًا.

## الخصوصية والبيانات المحلية

الأصل ألا تُرسل بيانات المستخدم إلى خادم. التخزين المحلي يجب أن يكون اختياريًا بعد موافقة واضحة، مع شرح مدة الاحتفاظ أو كون البيانات مرتبطة بالجلسة فقط. يجب أن يستطيع المستخدم سحب الموافقة بسهولة، وأن يوقف ذلك أي تخزين اختياري لاحق دون حذف صامت للبيانات القائمة قبل تحذيره.

يجب شرح التصدير والحذف الكامل والتحذير من إدخال أسماء أو بيانات تعريفية حساسة في الحقول الحرة. زر الحذف الشامل يجب أن يغطي جميع مفاتيح التخزين المحلية التي تملكها الأداة، وأن يكون آمنًا عند التكرار، وأن يؤكد النتيجة للمستخدم. عند فشل الحذف يجب عرض رسالة مفهومة وقابلة للوصول بدل ادعاء نجاح غير حقيقي.

ملفات الارتباط غير الضرورية لا تُفعّل قبل موافقة صريحة. لا يجوز إرسال مدخلات الأدوات النفسية الحساسة إلى التحليلات. إذا استُخدمت خدمة طرف ثالث، يجب تسميتها وشرح دورها قبل الإرسال.

## النتائج غير التشخيصية

لا يجوز للنتيجة أن تؤكد اضطرابًا أو تنفيه. يجب شرح معناها المحدود، وما يمكن للمستخدم فعله بعدها، ومتى يطلب مساعدة مهنية. لا يُعرض مسار الطوارئ إلا عندما يكون مستوى المحتوى عاجلًا أو توجد معايير تصعيد صريحة، وعندها يجب أن تكون الإرشادات محلية ومحددة دون اختلاق أرقام أو خدمات.

## قوة الدليل

يستخدم العقد تسميات واضحة: إرشاد رسمي، مراجعة منهجية، دراسة محكمة، إجماع مهني، خبرة عملية غير محسومة، أو دليل غير كافٍ. يجب عدم رفع مستوى الدليل بسبب شهرة المصدر أو تكرار الادعاء.

## الشفافية

لا يجوز ذكر خبير أو مراجع أو شريك أو اعتماد إلا إذا كانت الهوية والدور والعلاقة موثقة وقابلة للتحقق. يجب تسجيل التغييرات الجوهرية ومراجعة مصدر أي تعديل طبي أو نفسي مهم.
