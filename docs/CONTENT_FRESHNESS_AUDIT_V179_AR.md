# تدقيق حداثة المحتوى والمراجعة v179

## الهدف

يفحص المدقق سجلات المحتوى في `content/` و`data/`، وصفحات HTML المبنية في `_site/` عند توفرها، ويكشف الصفحات التي تبدو سليمة تقنيًا لكنها تفتقد تاريخ مراجعة أو حالة مراجعة واضحة أو مصادر منظمة حديثة.

## ما يفحصه

- حالة النشر والمراجعة.
- مستوى المخاطر الصحية.
- تاريخ آخر مراجعة وعمره بالأيام.
- وجود مصادر منظمة للمحتوى مرتفع أو حرج المخاطر.
- تاريخ تحقق المصادر وتقادمها.
- عنوان HTML ووصف الميتا وCanonical و`dateModified`.

## القرارات

- `current`: لا توجد فجوة مكتشفة وفق العقد.
- `needs-update`: الصفحة لا تُحذف تلقائيًا، لكنها تحتاج تحديث مراجعة أو مصدر.
- `fix-before-publish`: نقص جوهري يمنع اعتبار السجل جاهزًا للنشر، مثل محتوى صحي مرتفع المخاطر منشور بلا تاريخ مراجعة أو مصادر منظمة أو حالة مراجعة مكتملة.

## الحدود

هذا التدقيق لا يثبت صحة المعلومة الطبية، ولا يثبت أن المراجع اختصاصي حقيقي، ولا يغيّر حالة النشر، ولا يضع `noindex`، ولا يحذف صفحة، ولا يثبت ظهورها على الموقع الحي. مخرجاته قائمة عمل تحريرية تُراجع بشريًا.

## التشغيل

```bash
python scripts/audit_content_freshness_v179.py . \
  --output-json artifacts/content-freshness-v179.json \
  --output-csv artifacts/content-freshness-v179.csv \
  --today 2026-07-23
```

الإعداد الافتراضي يعتبر مراجعة المحتوى متأخرة بعد 365 يومًا، وتحقق المصدر قديمًا بعد 730 يومًا. يمكن تعديل الحدين من سطر الأوامر دون تغيير المحتوى.

## الاستخدام مع الناشرين المستقلين

يُوزع تقرير CSV على الناشر المسؤول عن القسم. لا يبدأ الناشر صفحة جديدة إذا كانت لديه صفحة مرتفعة المخاطر مصنفة `fix-before-publish`، ما لم يوجد سبب موثق لتقديم موضوع جديد أكثر إلحاحًا. يبقى الدمج والنشر خاضعين لبوابات المحتوى والسلامة والروابط وSEO والبناء والتحقق الحي.
